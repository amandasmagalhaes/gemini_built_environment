Comparison between virtual audits and Gemini

Published

May 5, 2026

1 Aim

To compare built environment characteristics collected through virtual audit and automated assessment using Gemini in a subsample of street segments from Belo Horizonte.

2 Virtual audit

The ELSI-Urbe Virtual Audit subproject was conducted using a systematic observation method between April and October 2024, based on available Google Street View images. The audit was performed at the street-segment level, considering both sides of the street, and later aggregated into census tracts, which were defined as neighborhoods. The sampling process was carried out in stages, sequentially selecting municipalities (the most populous municipality participating in ELSI-Brazil from each federative unit, with 20 available), census tracts (with ELSI-Brazil interviews), and random street segments. Data collection was conducted by a single trained rater using a multidimensional instrument and a field manual.

In the present analysis, we included only street segments from Belo Horizonte with at least five audited built environment characteristics available (n = 94 segments), namely: sidewalks, crosswalks, speed bumps, potholes, traffic lights, stop signs, sidewalk obstructions, bike lanes, trees, medians, lane markings, crossing signs, bus lanes, bus stops, and roundabouts.

3 Gemini

The same street segments included in the virtual audit were automatically processed using Google Street View images and a multimodal large language model (Gemini, Google) through a fully automated pipeline. For each coordinate, the closest panorama within a 10-meter radius and captured from 2015 onwards was selected, and the images were downloaded, preprocessed, and analyzed individually.

A prompt-based approach was used to classify the presence or absence of predefined characteristics. The model was instructed to consider only characteristics in the immediate vicinity of the camera and to account for distortions inherent to panoramic imagery. Outputs were generated as structured JSON objects with binary indicators (1 = present; 0 = absent or only visible at a distance).

In addition to the characteristics comparable to the virtual audit, the automated approach also evaluated a broader set of characteristics, including: curbs, streetlights, parked vehicles, yield signs, pedestrian signals, kiosks, bollards, median barriers, traffic signs, school zones, parking lanes, and BRT stations.

However, for the present analyses, only characteristics comparable to the virtual audit were considered, resulting in a final sample of 60 street segments with complete data from both approaches.

4 Statistical analysis

Descriptive frequencies were calculated for each characteristic in both approaches. Agreement between the virtual audit and the automated assessment was evaluated using observed agreement (%) and Cohen’s kappa coefficient, with corresponding 95% confidence intervals.

Kappa values were not estimated for characteristics with insufficient variability (i.e., when one or both approaches classified all observations into a single category), and were therefore reported as not applicable (NA).

The automated assessment using Gemini was implemented in Python. Data management and dataset organization were conducted in Stata, and all statistical analyses were performed in R.

5 Results

Agreement levels ranged from poor to almost perfect. Almost perfect agreement was observed for bus stop (n = 1), while substantial agreement was found for median, pedestrian crossing sign, traffic light, and roundabout (n = 4). Fair agreement was identified for crosswalk, speed bumps, potholes, and stop sign (n = 4), whereas slight and poor agreement were observed for lane markings (n = 1) and sidewalk obstruction (n = 1), respectively. No characteristics showed moderate agreement, and four variables had kappa not estimated due to insufficient variability: sidewalk, bike lane, trees, and bus lane.

calc_agreement <- function(var) {
  x <- df[[paste0(var, "_0")]]

}

# Robust kappa with correct CI extraction
calc_kappa <- function(var) {
  x <- df[[paste0(var, "_0")]]
  y <- df[[paste0(var, "_1")]]

  valid <- !is.na(x) & !is.na(y)
  x <- x[valid]
  y <- y[valid]

  # No variability → kappa undefined
  if (length(unique(x)) < 2 | length(unique(y)) < 2) {
    return(c(kappa = NA, lwr = NA, upr = NA))
  }
  k <- psych::cohen.kappa(cbind(x, y))
  ci <- k$confid[1, ]
  c(
    kappa = unname(ci["estimate"]),
    lwr = unname(ci["lower"]),
    upr = unname(ci["upper"])
  )
}

5.3 Diagnostic performance

Sensitivity and specificity, which reflect the ability to correctly identify the presence or absence of a characteristic using the field audit as the reference, were highest for more visible urban features such as trees, sidewalks, bus lanes, bike lanes, and roundabouts. Good results were also observed for medians, pedestrian crossing signs, bus stops, and traffic lights, whereas lower values were found for subtler attributes, including potholes, speed bumps, lane markings, and sidewalk obstructions.

Positive predictive value (PPV), which indicates the probability that a feature is truly present when classified as present, and negative predictive value (NPV), which indicates the probability that a feature is truly absent when classified as absent, were highest for trees, sidewalks, and bus stops, and lowest for lane markings, bus lanes, bike lanes, and sidewalk obstructions.

Similarly, F1-score, which summarizes the balance between sensitivity and positive predictive value, was higher for trees, sidewalks, and bus stops, intermediate for medians, traffic lights, pedestrian crossing signs, and crosswalks, and lower for the same lower-performing variables.

Diagnostic performance plots 
# Function to calculate all metrics

calc_metrics <- function(var) {
  ref <- df[[paste0(var, "_0")]]
  test <- df[[paste0(var, "_1")]]
  valid <- !is.na(ref) & !is.na(test)
  ref <- ref[valid]
  test <- test[valid]
  TP <- sum(ref == 1 & test == 1)
  TN <- sum(ref == 0 & test == 0)
  FP <- sum(ref == 0 & test == 1)
  FN <- sum(ref == 1 & test == 0)
  tibble(
    variable = var_labels[var],
    sensitivity = ifelse((TP + FN) == 0, NA, TP / (TP + FN)),
    specificity = ifelse((TN + FP) == 0, NA, TN / (TN + FP)),
    ppv = ifelse((TP + FP) == 0, NA, TP / (TP + FP)),
    npv = ifelse((TN + FN) == 0, NA, TN / (TN + FN)),
    f1_score = ifelse((2 * TP + FP + FN) == 0, NA, 2 * TP / (2 * TP + FP + FN))
  )
}

5.4 ROC

Receiver operating characteristic (ROC) analyses, which evaluate discriminatory ability by comparing sensitivity with the false positive rate, confirmed these patterns. Higher area under the curve (AUC) values were observed for roundabouts, medians, pedestrian crossing signs, bus stops, and traffic lights, whereas crosswalks, stop signs, potholes, and speed bumps showed intermediate performance. Lane markings and sidewalk obstructions presented the weakest discrimination.

ROC panel 
# Function to calculate ROC object + AUC
calc_roc_data <- function(var) {
  ref <- df[[paste0(var, "_0")]]
  pred <- df[[paste0(var, "_1")]]
  valid <- !is.na(ref) & !is.na(pred)
  ref <- ref[valid]
  pred <- pred[valid]
  if (length(unique(ref)) < 2) {
    return(NULL)
  }
  roc_obj <- roc(ref, pred, quiet = TRUE)
  coords_df <- tibble(
    specificity = roc_obj$specificities,
    sensitivity = roc_obj$sensitivities
  ) %>%
    mutate(
      fpr = 1 - specificity,
      variable = var_labels[var],
      auc = as.numeric(auc(roc_obj))
    )
  coords_df
}

Overall, across all agreement and diagnostic metrics, Gemini consistently performed better for larger, permanent, and visually salient urban features, such as bus stops, medians, pedestrian crossing signs, traffic lights, and roundabouts. Intermediate performance was observed for features like crosswalks, stop signs, speed bumps, and poor street condition (potholes). In contrast, lower performance was found for more subtle, linear, or condition-dependent characteristics, including lane markings and sidewalk obstructions. These findings suggest that multimodal artificial intelligence may be a useful tool for automated urban audits of clearly visible built-environment features, although caution is warranted for more ambiguous street attributes.

6 Discrepancy analysis

In this step, discrepancies between human and AI assessments are examined for features selected due to their intermediate and lower performance in agreement and diagnostic metrics. For each variable, segments with disagreement are identified, the type of discrepancy is classified (human = 1 and AI = 0; or human = 0 and AI = 1), and these cases are linked to geographic coordinates and Google Street View links, enabling visual inspection and interpretation of the differences.

# Function: capture BOTH discrepancy types
get_ids_discrepancy <- function(data, base_var) {
  var0 <- paste0(base_var, "_0")  # human
  var1 <- paste0(base_var, "_1")  # AI
  
  data %>%
    dplyr::filter(!is.na(.data[[var0]]),
                  !is.na(.data[[var1]]),
                  .data[[var0]] != .data[[var1]]) %>%
    dplyr::mutate(
      discrepancy_type = dplyr::case_when(
        .data[[var0]] == 1 & .data[[var1]] == 0 ~ "human_1_ai_0",
        .data[[var0]] == 0 & .data[[var1]] == 1 ~ "human_0_ai_1"
      )
    ) %>%
    dplyr::select(identificador, discrepancy_type) %>%
    distinct()
}
